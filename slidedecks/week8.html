Important Terms

DBMS: database management system, sometimes RDBMS for relational DBMS

Type: the data type, such as integer, string/text, etc.

Domain: the acceptable values for attributes

Entity: the collection of related data items that are treated as a unit (e.g. roads, lakes)

Instance: a specific instance of an entity (e.g. Colorado Blvd, Lake Arrowhead)

Tuple: technical term that can be defined a number of ways; think of it as a record or a set of ordered values. Sometimes referred to as n-tuple

Rows and Columns

Rows: records or tuples

Columns: attributes or items

More Terms

Super key - one or more attributes that may be used to uniquely identify every record for a table

Candidate key - a minimal super key

Primary key - the single candidate key chosen to index the table

Functional dependency - the relationship between two attributes where one necessarily determines the other
